6.  DATABASE TABLES (Supabase Migrations)
================================================================================

--- Migration 001 — events -----------------------------------------------------

  Column          Type            Constraints
  ──────────────────────────────────────────────────────────────────────────────
  id              UUID            PK, default gen_random_uuid()
  organizer_id    UUID            NOT NULL
  name            TEXT            NOT NULL
  description     TEXT            nullable
  start_date      DATE            NOT NULL
  end_date        DATE            NOT NULL
  venue           TEXT            nullable
  category        TEXT            default 'General'
  max_attendees   INT             default 500
  status          TEXT            default 'Draft'
  created_at      TIMESTAMPTZ     default now()

--- Migration 002 — Sponsorships & Certifications ------------------------------

  Table                    Key Columns                                            Relationships
  ─────────────────────────────────────────────────────────────────────────────────────────────────
  event_sponsors           id, event_id (FK->events), name, tier (CHECK),         CASCADE delete
                           amount, contact_email, pipeline_stage (CHECK)          with events

  sponsor_inquiries        id, company, contact_name, email,                      Standalone
                           interested_tier, message                               (public insert)

  certificate_batches      id, event_id (FK->events, UNIQUE), event_name,         CASCADE delete,
                           conducted_date, template_data_url                      unique per event

  event_certificates       id, batch_id (FK->batches), event_id (FK->events),     CASCADE delete
                           participant_name                                       via batch or event

  RLS Policy pattern on all organizer tables:
    EXISTS (SELECT 1 FROM events WHERE events.organizer_id = auth.uid())
